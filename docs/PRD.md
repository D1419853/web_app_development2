# 追劇推薦系統 PRD (Product Requirements Document)

## 1. 專案概述

### 背景與動機
隨著影視串流平台（如 Netflix, Disney+, YouTube）的普及，劇集數量呈爆炸式增長。觀眾經常面臨「劇荒」的困擾，花費大量時間在各平台間切換卻不知該看哪部劇。本系統旨在建立一個集中的推薦平台，讓使用者能快速找到感興趣的內容。

### 目標用戶
- 喜愛追劇但面臨選擇困難的觀眾。
- 想尋找特定演員或類型作品的影視愛好者。
- 追求高品質（高評分）內容的資深影迷。

### 核心價值主張
「告別劇荒，一秒找到你的命定劇。」—— 透過多維度的分類與客觀的評分機制，提供效率與品質兼具的選劇體驗。

---

## 2. 功能需求

| 功能名稱 | 使用者故事 (User Story) |
| :--- | :--- |
| **演員分類** | 作為觀眾，我希望可以根據演員名稱篩選劇集，以便追蹤心儀藝人的所有作品。 |
| **類型分類** | 作為觀眾，我希望可以根據題材（如：懸疑、愛情、科幻）過濾清單，以便根據當下心情選劇。 |
| **觀眾評分** | 作為觀眾，我希望看到每部劇集的平均評分，以便在觀看前評估是否符合期待，避免踩雷。 |
| **熱門劇集榜單** | 作為觀眾，我希望看到系統整理的 Top 10 熱門榜單，以便快速了解目前流行的話題作品。 |
| **個人收藏清單** | 作為觀眾，我希望可以將感興趣的劇集加入「想看清單」，以便日後有空時快速取出觀看。 |

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask。
- **模板引擎**：使用 Jinja2 進行動態頁面渲染。
- **資料庫**：使用 SQLite 存儲劇集、演員、評分與用戶清單資訊。

### 效能與安全考量
- **響應速度**：頁面跳轉與分類篩選應在 1 秒內完成。
- **資料安全性**：確保 SQLite 資料庫查詢使用參數化查詢，防止 SQL 注入。
- **介面美觀**：遵循現代 Web 設計美學，使用 CSS 提供流暢的互動體驗。

---

## 4. MVP 範圍

### Must Have (核心功能)
- 劇集資料庫建立（包含名稱、海報、演員、類型）。
- 依演員、類型進行篩選的功能。
- 顯示觀眾評分。

### Should Have (重要功能)
- 全站關鍵字搜尋功能。
- 個人「想看清單」收藏功能。
- 劇集詳細介紹頁面。

### Nice to Have (進階功能)
- 使用者留言與評論系統。
- 根據用戶喜好自動推薦相似劇集。
- 串流平台跳轉連結（例如：直接跳轉至 Netflix 播放頁面）。
